# App Store 掲載文言（概要・最新情報）

App Store Connect (ASC) に入力する掲載文言（「このバージョンの最新情報 / What's New」
「概要 / Description」「プロモーションテキスト / Promotional Text」）の記載方針・文字数上限を
定め、提出ごとに用意した文面をスナップショットとして台帳化するためのテンプレート兼運用ガイド。
リリースのたびに同じトーンで再現でき、取りこぼし・トーンのばらつきを防ぐことを目的とする。

> **アプリ固有値**: `{{app_name}}` 等のプレースホルダは、各アプリの
> `.claude/flutter-ios-profile.md` の値に読み替える。
>
> 本ファイルは雛形であり、台帳（各バージョンの実テキスト）は **各アプリで記入する**。
> 以下の「記入例」はローカル完結アプリの典型例であり、実装・トーンに応じて差し替える。

## Source of Truth

- **正本**: App Store Connect（[appstoreconnect.apple.com](https://appstoreconnect.apple.com)）の本アプリレコード
- **本ファイル**: 上記に入力した掲載文言を Markdown で写したスナップショット台帳。一次情報ではない

ASC web UI が正本であり、本ドキュメントはそのスナップショット。掲載文言を変更する際は
ASC 側を更新したうえで、本ファイルの台帳も同期する。文言の生成・台帳同期は
`/release-ios-build` スキルの掲載文言ステップが担う。

## 対象フィールドと文字数上限

ASC の「バージョン情報」「App 情報」で入力する各フィールドと上限。上限は ASC の制約に従う。

| フィールド | ASC 上の名称 | 文字数上限 | 更新頻度 |
| --- | --- | --- | --- |
| 最新情報（What's New） | このバージョンの最新情報 | 4,000 文字 | バージョンごとに必ず更新 |
| 概要（Description） | 概要 | 4,000 文字 | 主要機能の変化時のみ更新 |
| プロモーションテキスト | プロモーションテキスト | 170 文字 | 任意（版の提出なしで差し替え可） |

- キーワード（参考: 170 文字）は本ガイドの記載方針の対象外（検索最適化の別観点）。
- 文字数は ASC のカウント基準（改行・空白含む）に従う。上限超過は ASC 側で保存できないため、
  ドラフト生成時に超過していないか必ず確認する。

## What's New 記載方針

「このバージョンの最新情報」は次の方針で書く。

- **ユーザー向け変更のみ**を載せる。tech-debt 解消 / CI / 内部リファクタ / ドキュメント等の
  非ユーザー向け変更は載せない。
- `■` を見出し記号に用い、平易な日本語で簡潔に書く。
- **未実装・将来予定の機能には触れない**（提出版で実際に使える変更のみ）。
- 修正（バグ修正）と新機能・改善を区別して並べる。ユーザー価値の高いものを上に。

### 記入例

```text
■ 新機能
・〇〇を追加しました。△△できるようになります。

■ 改善
・□□の表示を見やすくしました。

■ 修正
・特定の条件で◇◇が正しく動作しない問題を修正しました。
```

## 概要 / プロモーションテキストの方針

- **概要（Description）**: 全文の書き換えではなく、今回の新機能が主要機能に該当する場合のみ
  **追記差分**を提案する。既存の構成・トーンを維持する。
- **プロモーションテキスト**: 任意。キャンペーンや直近の目玉機能に合わせた差し替え案を、
  170 文字以内で提案する（不要なら据え置き）。

## ASC スナップショット台帳（記入例）

提出ごとに、ASC に入力した実テキストを以下の形式で追記する（最新を上に）。

### v0.0.0（YYYY-MM-DD 提出）※記入例

**このバージョンの最新情報**

```text
■ 改善
・（実際に ASC へ入力した文面をそのまま記録する）
```

**概要 追記差分**（あれば）

```text
（今回追記した段落のみ。変更なしの版では「変更なし」と記す）
```

**プロモーションテキスト**

```text
（差し替えた場合のみ記録。据え置きなら「据え置き」と記す）
```
