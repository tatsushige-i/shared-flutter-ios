# App Privacy（収集データ申告）

App Store Connect (ASC) の「App のプライバシー (App Privacy)」セクションで自己申告する
収集データ・トラッキングの内容を整理するためのテンプレート兼運用ガイド。提出・再申請時に
迷わず入力できるよう、各アプリで本ファイルを埋めて台帳化する。

> **アプリ固有値**: `{{app_name}}` / `{{privacy_policy_url}}` 等のプレースホルダは、各アプリの
> `.claude/flutter-ios-profile.md` の値に読み替える。
>
> 本ファイルは雛形であり、申告結論や根拠の表は **各アプリの実装に合わせて記入する**。
> 以下の例示行（drift / オンデバイス SQLite 等）はローカル完結アプリの典型例であり、
> ネットワーク通信・解析・課金等を持つアプリでは内容が変わる。

## Source of Truth

- **正本**: App Store Connect（[appstoreconnect.apple.com](https://appstoreconnect.apple.com)）の本アプリレコード
- **本ファイル**: 上記の申告判断を Markdown で写したサマリ。一次情報ではない

ASC web UI が正本であり、本ドキュメントはそのスナップショット。
変更が発生したら ASC 側で更新したうえで、本ファイルも同期する。

## 申告結論（記入例）

| 項目                  | 値                                              |
| --------------------- | ----------------------------------------------- |
| データ収集 (Data Collection) | **Data Not Collected（収集データなし）** ※要判定 |
| トラッキング (Tracking)      | **なし**（ATT プロンプトなし） ※要判定         |

Apple の定義では「ユーザーの端末内にのみ保存され、外部（自社サーバー・第三者）へ送信されない
データ」は *collect* に該当しない。記録データがすべて端末内のローカル DB に保存され外部送信が
一切無いアプリは `Data Not Collected` を選択できる。外部送信・解析・広告 SDK 等を持つ場合は
収集データタイプごとに用途・リンク有無・トラッキング有無を申告する。

## 根拠（実装で確認すべき観点）

申告結論を裏付けるために、各アプリの実装で以下を確認する。

- **永続化の経路**: ローカル DB のみか、外部ストレージ・クラウド同期を持つか
  （例: drift によるオンデバイス SQLite に保存しており外部ストレージが無い、等）
- **ネットワーク・計測コードの有無**: `lib/` 配下に `package:http` / `HttpClient` /
  Firebase / Analytics / Crashlytics / 広告 SDK 等のネットワーク送信・計測コードが存在するか
- **センシティブ権限の宣言**: `ios/Runner/Info.plist` に `NS*UsageDescription`
  （カメラ・位置情報・連絡先等）や `NSUserTrackingUsageDescription`（ATT）の宣言があるか
- **ユーザー生成の公開・共有機能の有無**: 記録が本人閲覧のみか、外部公開・他ユーザー共有を行うか

## 使用パッケージの外部送信有無（記入例）

ランタイムで動作するパッケージごとに外部送信の有無を確認し、表に整理する。

| パッケージ                          | 用途                       | 外部送信 |
| ----------------------------------- | -------------------------- | -------- |
| `flutter_riverpod` / `riverpod_annotation` | 状態管理                | なし     |
| `go_router`                         | 画面遷移                   | なし     |
| `drift` / `drift_flutter`           | ローカル SQLite 永続化     | なし     |
| `cupertino_icons`                   | アイコンフォント           | なし     |

`build_runner` / `riverpod_generator` / `freezed` / `drift_dev` / `flutter_launcher_icons` 等は
**ビルド時の dev 依存**であり、アプリのランタイムには含まれない（データ収集に非関与）。

## ASC 操作手順

ASC web UI 上で App Privacy を申告する際の操作手順。再申請・別アプリ展開・引き継ぎ時に
同じ判断を再現するためのガイド。

- ASC ナビゲーション: 対象アプリ → サイドバー「App のプライバシー (App Privacy)」 → 「編集 (Edit)」
- データ収集の質問: 収集なしなら **「No, we do not collect data from this app」を選択**する
  （根拠は本ファイル「根拠」セクション参照）
- 上記を選択すると、データタイプ別の詳細入力（用途・リンク有無・トラッキング有無）は不要になる
- **入力後、画面右上の「公開 (Publish)」ボタンを押して確定する**。公開しないと下書き状態のままで、
  版（バージョン）ページの「審査用に追加」がプライバシー未完エラーでブロックされる
- 注: Apple 側でデータカテゴリや質問文は時折更新されるため、本 docs では方針のみ記録し、
  個別データタイプの全列挙は意図的に行わない

## プライバシーポリシーとの整合

- Privacy Policy URL: `{{privacy_policy_url}}`
- プライバシーポリシー本文の作成は別作業。本台帳の申告とポリシー本文の記載を
  矛盾させないこと（例: 「収集なし」と申告するならポリシー側でも「データを収集・外部送信しない」
  旨を明記する）。
